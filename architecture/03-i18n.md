# 03 — Internationalization (i18n)

The UI ships **English only** now. French (and others) are added later by dropping a JSON
file in — **no code change required**. The language switcher reflects available locales
dynamically.

## Locale files

```
packages/web/src/i18n/locales/
├─ en.json   # shipped now (source of truth for keys)
└─ fr.json   # added later by the author
```

Each file is a flat or shallow-nested key→string map:

```json
{
  "nav.newspapers": "Newspapers",
  "events.nextIn": "Next event in {time}",
  "common.save": "Save"
}
```

- **`en.json` is the canonical key set.** Other locales mirror its keys; missing keys
  fall back to English.
- Interpolation uses `{placeholders}`.

## Auto-discovery (no manual registry)

Vite globs the folder so adding a file is enough:

```ts
// i18n/index.ts
const files = import.meta.glob('./locales/*.json', { eager: true });
// → { './locales/en.json': {default: {...}}, './locales/fr.json': {...} }
export const locales = Object.fromEntries(
  Object.entries(files).map(([path, mod]) => [
    path.match(/\/(\w+)\.json$/)![1],      // 'en', 'fr'
    (mod as any).default,
  ]),
);
export const availableLocales = Object.keys(locales); // drives the switcher
```

A tiny `t(key, vars?)` helper reads the active locale, falls back to `en`, and substitutes
`{vars}`. A `<LanguageProvider>` holds the active locale (persisted to `localStorage`,
defaulting to the browser language if available, else `en`).

> Keep it dependency-light: a ~30-line provider + `t()` is enough. A library
> (`react-i18next`) is acceptable but not required.

## Rules for contributors

1. **Never hard-code user-facing text** in components — add a key to `en.json` and use `t()`.
2. Server error `code`s are also i18n keys (e.g. `error.newspaper.pending`), translated
   client-side. Keep them stable.
3. To add a language: copy `en.json`, translate values, save as `<lang>.json`. Done.

## Testing (Vitest)

- A test asserts every non-English locale has **no keys missing** vs `en.json`
  (and warns on extra keys), so translations stay in sync as keys are added.
